
# AES-256-GCM 文件加密工具

考虑到 age、gpg、openssl 等软件可以非常好地满足非对称/对称加密需求，但似乎没有软件满足让用户直接使用 AES-256-GCM 的需求，更没有软件可以解决用户使用 123456 等密码的问题。所以，本软件应运而生。

## 特性

**一**：只支持 AES-256-GCM 加密，达到极致的加密能力  
**二**：自动生成密钥和 IV，防止简单密码  
**三**：极致简单，C 语言 + OpenSSL 编写，极致速度

## 编译

需要 OpenSSL 开发库（libssl-dev）。

使用项目提供的编译脚本：

```bash
bash .build.sh
```

或手动编译：

```bash
gcc enc_hand.c -o enc_hand -O2 -Wall -Wextra -lcrypto
```

## 用法

```bash
./enc_hand <输入文件> <密文输出> <元数据输出>
```

**参数说明：**
- `<输入文件>`：待加密的明文文件
- `<密文输出>`：加密后的密文文件
- `<元数据输出>`：保存密钥、IV、TAG 的二进制文件（解密时需要）

**示例：**

```bash
# 加密文件
./out plain.txt cipher.bin meta.bin

# 查看输出
ls -lh cipher.bin meta.bin
```

## 工作原理

1. 自动生成 256-bit 随机密钥和 128-bit 随机 IV
2. 使用 AES-256-GCM 模式加密输入文件
3. 密文写入 `<密文输出>`
4. 密钥、IV、TAG（认证标签）二进制写入 `<元数据输出>`

## 安全注意

- 每次加密自动生成全新的随机密钥和 IV，确保安全性
- GCM 模式提供加密和认证双重保护
- **请妥善保管元数据文件**，丢失后密文无法解密
- 元数据文件包含明文密钥，需要与密文分开安全存储
